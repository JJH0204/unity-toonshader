### 2-1. 그림자 얼룩 해결 + 그림자 UI 통합 + 마스크

**현상**

**원인 가설 (회의 확인)**
1. Receive Shadow ON 자체가 얼룩 주원인.
2. URP Normal Bias 값으로 노말 방향 일그러짐.

### 2-2. 노말맵 적용 방식 수정

**현상**
- 노말맵이 카메라(시점) 방향에 반응. 카메라 회전 시 디테일이 따라 움직이고 일정 거리 벗어나면 사라짐 → 프레넬/림처럼 동작.

**현상 (2026-06-19 추가)**
- 노말맵 적용 메시를 **멀리서 보면 노말 디테일이 보이는데, 가까이 다가가면 사라진다.** (회의 시 "카메라 따라 움직임/일정 거리 벗어나면 사라짐"과 동일 계열 증상)

**근본 원인 — 확정 (2026-06-19, Claude 코드 분석)**
- N은 섭동 노멀이 맞고(`CharacterToon.shader:204-208`) 디퓨즈 `ndotl`·SH·림·MatCap 모두에 전파됨 → "디퓨즈 미반영 버그" 아님.
- **진짜 원인 = `ApplyNormalMapDerivative`(ToonLighting.hlsl)의 퇴화 가드 임계값 `if (maxLenSq < 1e-12) return N;`이 근거리에서 정상 프레임을 오판해 노멀맵을 꺼버림.**
  - 코탄젠트 탄젠트 `T ≈ ddx(posWS) × ddx(uv)` 크기는 **픽셀 풋프린트의 제곱**에 비례. 근거리일수록 ddx(posWS)·ddx(uv) 둘 다 작아져 `dot(T,T)`가 제곱으로 급감.
  - 추정 크기: 원거리(객체 ~100px) `dot(T,T)≈7e-9`(통과) vs 근거리(객체 ~1080px, 0.5m) `≈1.8e-13`(< 1e-12 → `return N`, 섭동 꺼짐).
  - 결과의 *방향*은 스케일 불변(invmax 정규화)인데, **절대 임계값**만 스케일 종속이라 근거리 드롭아웃 발생. 증상과 정확히 일치.
- 부차 원인(회의의 "카메라 추종" 인지): 셀 단계화 디퓨즈는 평탄 영역에서 미세 섭동을 못 보여주고, 뷰 종속항(Rim fresnel/MatCap)에서만 디테일 표출 → 원거리(그레이징 많음)에서 림으로 보이고 근거리(정면)에서 약해짐. 1차 원인 수정 후 잔존 시 별도 대응.
- 대안 가설(여전히 점검): 텍스처가 Texture Type=Normal map 미설정(PNG 캡처) → 미적용.

**수정안 (1차 원인)**
- 절대 임계값 early-return 제거 → 0-나눗셈만 막는 near-zero 플로어로 교체(스케일 불변 보장):
  `float invmax = rsqrt(max(maxLenSq, 1e-30));` (early-return 삭제). 진짜 0 프레임이면 T·B=0 → 결과 ≈ N로 자연 폴백, NaN 없음.
- 키워드 off no-op·미러 UV·스키닝 정합은 그대로. Codex 재검증 + 사용자 근접 확인.

**진단 절차 (사용자 — Unity, 헤드리스 불가)**
> 새로 추가된 2-4 Use 토글로 항을 분리 관측할 수 있다.
0. **임포트 확인**: `_BumpMap` 텍스처 Inspector → Texture Type = **Normal map**, sRGB OFF. `Use Normal Map` 토글 ON. `Normal Scale`를 2 근처로 올려 과장.
1. **1차 원인 확인(가장 유력)**: 메시에 **가까이** 다가간다. 디테일이 사라지면 → 라이트만 회전해도 근거리에선 음영 변화 없음(=섭동 꺼짐). **이 상태에서 수정안 적용 후 다시 근접** → 디테일 유지되면 1차 원인 확정.
2. **뷰 종속 분리**: `Use Rim Light` OFF + `Use Depth Rim` OFF + (MatCap 쓰면)`Use MatCap` OFF. 원거리에서도 디테일이 사라지면 → 그건 림/MatCap(뷰 종속)이 보여주던 것. 디퓨즈만 남김.
3. **디퓨즈 응답(모델러가 원하는 빛 기준)**: 위 상태에서 **디렉셔널 라이트만** 회전해 명암 경계(터미네이터)가 굴곡 부위를 쓸도록 한다. 경계선이 굴곡 따라 **꿈틀거리면** 노멀이 디퓨즈에 정상 반영(셀 양자화로 평탄부는 원래 안 보임 — 툰 한계).
4. **카메라 vs 라이트**: 라이트 고정·카메라만 회전 → 디테일이 카메라 따라 돌면 안 됨(돌면 뷰 종속 잔존). 카메라 고정·라이트만 회전 → 굴곡 따라 음영 변해야 정상.
5. **셀 양자화 확인(부차)**: `Shadow Blur`를 키워 경계를 부드럽게 → 디퓨즈에서 굴곡 그라데이션이 보이면, 평탄부 디테일이 안 보이던 건 하드 셀 때문(버그 아님).

**완료 기준**
- **근거리에서도 노멀 디테일 유지**(1차 원인 수정). 카메라 회전 시 디테일이 따라 돌지 않음. 라이트 이동 시 굴곡 따라 음영 변화.

**회귀 영향**: WP-B no-op 보장(키워드 off) 유지. §5 Normal Map 항목.

**진행 (2026-06-19, Claude 수정 + Codex 4/4 PASS)**
