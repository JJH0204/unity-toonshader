## M0 — URP 빈 프로젝트 + NiloCat 포팅 스파이크
목적: Unity 6.3/URP 17 실제 경로 확정. 결정 #1·#2·#6·#8을 닫는다. (11장 M0)

- [~] **T0-1** 현재 프로젝트가 URP 17.3로 정상 렌더되는지 베이스라인 확인 (PC/Mobile RP 에셋 양쪽, SampleScene)
  - 렌더러 설정 확인됨: **PC_Renderer = Forward+ (RenderingMode 2) + SSAO**, **Mobile_Renderer = Forward (0)**, Deferred 미사용
  - 남은 것(에디터): SampleScene에서 머티리얼 시각 확인 + 셰이더 에셋 Inspector의 "SRP Batcher: compatible" 눈으로 확인
- [x] **T0-2** NiloCat 포팅 여부 → **결정 #2 확정: 자작.** 자작 스켈레톤이 6.3/URP17.3에서 무에러 임포트 + 배쳐 호환(Codex 교차검증). NiloCat은 기법 참고만, 포팅 베이스 아님
- [x] **T0-3** ForwardToon LightMode → **결정 #1 확정: `UniversalForwardOnly`.** Deferred 미사용이나 GBuffer 패스 없는 캐릭터 셰이더 → 세 경로 모두 안전. 셰이더 반영 완료
- [x] **T0-4** Forward+ additional light 경로 → **결정 #8 확정: 셀 라이트는 글로벌 프로퍼티 주입 + 씬 additional light는 Forward+ 클러스터 루프 병행.** URP 17.3 키워드 `_CLUSTER_LIGHT_LOOP`(구 `_FORWARD_PLUS` deprecated) 추가, 누락 버그 수정 완료
- [x] **T0-5** 아웃라인 보정식 → **결정 #6 방향 확정: clip-space w 스케일로 화면상 두께 일정화.** 프로토타입 셰이더 반영. 최종(FOV항·스무스노멀·비균일스케일)은 M3
- [~] **T0-6** 최소 `CharacterToon.shader` 스켈레톤 작성: 5개 패스 스텁 + `CBUFFER_START(UnityPerMaterial)` 골격 → **SRP Batcher 호환(밴딩) 확인**
  - 작성 완료: `Assets/CharacterToon/Shaders/CharacterToon.shader`, `Assets/CharacterToon/ShaderLibrary/ToonInput.hlsl` (전 패스 단일 CBUFFER 공유)
  - 남은 것: Unity 에디터에서 컴파일 통과 + Inspector의 "SRP Batcher: compatible" 확인 (= T0-1과 함께 수행)
- [ ] **(게이트)** 컴파일·씬 렌더·SRP Batcher 호환 확인, 결정 #1/#2/#6/#8 문서에 기록

> M1 구현: Copilot 로컬 위임 → Codex 검증 통과 → 통합. (Copilot=구현, Codex=검증, Claude=통합/수정)
- [x] **T1-1** BaseMap 샘플링 + `_BaseColor` 적용 ForwardToon 본문 구현
- [x] **T1-2** ILM 샘플링 + 채널 규약 적용, 누락 시 중립값 `(0,0.5,0.5,0)` 폴백. `[Toggle(_USE_ILM)]` 키워드로 토글 (3.2장)
- [x] **T1-3** Ramp 음영: half-Lambert + `shadowBias=(ilm.g-0.5)*_ShadowOffsetScale` → `rampU`, `_RampRow` (4.1장)
- [x] **T1-4** 밝기 하한 + 앰비언트: `max(shaded, base*_ShadeFloor)` + `SampleSH(N)*_AmbientStrength` (4.3장)
- [x] **T1-5** 캐릭터 라이트 주입: `CharacterToonLight.cs`(글로벌 `_CharacterLightDirWS`, w=1/0 폴백). Runtime asmdef 생성. **OnDisable에서 글로벌 클리어**(Codex 지적 stale 버그 수정) (4.2·8장)
- [x] **T1-6** 림 라이트: fresnel→smoothstep, `LOBBY_HQ` 시 `_RimInteractionBoost` (4.4장). 상호작용 스크립트 훅은 M4에서 연결
- [ ] **T1-7** 다수 캐릭터 라이트 방식 1차 검증: 메인추종 vs 라이트그룹 vs StructuredBuffer+ID → **결정 #4** 착수 (에디터 1체/10체 실측 필요)

## M2 — Face SDF + 얼굴 검증 툴 최소판
목적: 근접 얼굴 음영 품질. (11장 M2 / 본문 5·9장)

> M2 구현: Copilot 위임 → Codex 검증(NaN 버그 1건 지적) → Claude 통합/수정.
- [x] **T2-1** Face SDF 플립 샘플링: `_PART_FACE`에서 `FdotL`/`RdotL`로 좌우 SDF 선택, `coverage`, `LOBBY_HQ`=smoothstep/인게임=step, `rampU=faceLit` (5장). **버그 수정**: 라이트 폴백 시 `normalize(0,0)` NaN → `L.xz` 안전 정규화로 교체(Codex 지적)
- [x] **T2-2** `CharacterToonFace.cs`: `_FaceForwardWS`/`_FaceRightWS` 글로벌 주입(Transform forward/right, w=1)
- [x] **T2-3** **Face SDF 검증 툴** `FaceSdfValidatorWindow.cs` (메뉴 Window/CharacterToon/Face SDF Validator): 광원 방위각 슬라이더 + Front/Left/Right/Back 프리셋 + forward/right 벡터 적용 + SceneView 갱신. `CharacterToon.Editor.asmdef`(Editor 한정, Runtime 참조) 생성 (9장 #2)
- [x] **T2-4** SDF 흑백·UV 대칭·플립 방향 → **결정 #5 확정: 현재 구현대로 정확.** 검증 툴 실측: 정면=밝음/후면=그림자(coverage 방향 정상), ±45° 측면광에서 빛 반대편이 그림자(플립 방향 정상), 볼 음영은 SDF 텍스처 의도대로. 셰이더 반전 불필요

- [x] **T2-5** 헤어 그림자 1단계: `_HairShadowMask` + `_HairShadowStrength` + `_USE_HAIR_SHADOW` 키워드, `_PART_FACE`에서 faceLit 감쇄 (5장)
- [ ] **T2-6** 헤어 그림자 자동 투영 → **결정 #7**: 수동 마스크만 구현, 프록시 투영은 M2 범위 밖으로 보류(방향 메모 유지)
- [~] **(게이트)** 좌/우/정면/후면 라이트 **통과(셰이더 메커니즘·플립·coverage 검증 완료)**. 로비 클로즈업 "품질" 도달은 실제 얼굴 메시 + 하드 2단 RampMap 셋업 후 아트 판단(M4 룩 작업 시 최종 확인)

## M3 — Inverted Hull 아웃라인 + 스무스 노멀 베이크 연동
목적: 거리/FOV/스키닝에 안정적인 외곽선. (11장 M3 / 본문 6·9장)

> M3 구현: Copilot 위임 → Codex 검증(베이커 치명 버그 2건 지적) → Claude 통합/수정.
- [x] **T3-1** Outline 패스: `Cull Front`, tangent.xyz 스무스 노멀 푸시(폴백: tangent 0이면 메시 노멀), width=0 시 오프셋 0 (6장)
- [x] **T3-2** 스무스 노멀 저장 채널 → **결정 #3 확정: tangent.** (Unity가 스키닝 시 tangent 변형 → 스킨드 아웃라인 유리. M4 Angel Ring tangent는 추후 별도 소스 분리 필요 — 보류)
- [x] **T3-3** **스무스 노멀 베이커** `SmoothNormalBaker.cs` (Window/CharacterToon/Smooth Normal Baker + Assets 컨텍스트 메뉴): 위치 기준 노멀 평균 → tangent.xyz, FBX 무수정 새 메시 에셋 생성. **버그 수정**: 서브메시 보존(subMeshCount+SetTriangles), indexFormat 복사(>65535 정점) — 둘 다 Codex 지적 (9장 #1)
- [x] **T3-4** 거리/FOV 보정 → **결정 #6 최종 확정**: `widthWS = _OutlineWidth*0.01*positionCS.w / UNITY_MATRIX_P._m11` (거리: ×w / FOV: ÷cot(fov/2) / 비균일스케일: inverse-transpose 노멀변환)
- [x] **T3-5** 불필요 선 억제: `_USE_ILM` 시 Outline 정점에서 ILM.A 샘플 → width ×(1-ilm.a)로 국소 제거 (6장)
- [x] **T3-6** 아웃라인 LOD: width=0이 외곽선 완전 제거 → 군중/LOD는 width 0 머티리얼 프리셋으로 비활성
- [ ] **(게이트)** 스키닝·비균일 스케일·FOV에서 외곽선 안정 (에디터 실측: 베이커로 메시 베이크 후 확인)
  - 신규: `SmoothNormalBaker.cs`

- [x] **T4-3** `_PART_SKIN` SSS 근사: `rampU` 경계 따뜻한 색 가산 (7장)
