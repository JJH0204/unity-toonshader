> 출처: 모델러 팀(lilToon 숙련) 필수 기능 피드백(2026-06-18). 메모리 [[modeler-feature-requirements]].
> 원칙: **lilToon은 게임 출하에 부적합 → 모델러가 필요로 하는 lilToon 기능을 게임 최적화 셰이더로 재현**한다(자작 HLSL, lilToon 코드 비참조).

## B. 작업 패키지(WP) — "#2" 구현 범위
> 각 WP: 셰이더(ToonInput/CharacterToon) + ShaderGUI 라벨/툴팁 + Codex 검증. (이 디렉터리 Copilot 쓰기 불가 → Claude 직접 + Codex)

- **WP-A 음영 모델 정비 (P0, 최우선·토대)** — 1·2차 그림자 밴드(border/blur/color), 적용 범위, 그림자↔빛 경계, SDF 컬러. → **결정 #17** 선행.
- **WP-B 노멀 맵 (P0)** — `_BumpMap`/`_USE_NORMALMAP`, 탄젠트 공간 N, 라이팅·MatCap·스페큘러에 반영.
- **WP-C MatCap 정비 (P0~P1)** — 별도 `_MatCapMask`(결정 #16), 노멀 기반 UV(#11), 가산/Emissive 블렌드(#3). 상시 활성(#12 완료).
- **WP-D PBR 반사 (P1)** — metallic/roughness 맵 + reflection probe(`GlossyEnvironmentReflection`), 툰 단계화. 옷/금속 구분.
- **WP-E 아웃라인 정비 (P1)** — 외곽선 텍스처 색 + 전용 마스크(#6), 거리 굵기 감쇠(#7, 현 화면일정식에 거리 페이드 옵션).
- **WP-F 테셀레이션 (P2, 결정 #18)** — Hull/Domain 패스. 거리/품질 게이트(S2) 전제, 모바일 비용 점검.

권장 순서: **A → B → C → E → D → F** (A가 룩 토대, F는 마지막·선택).

## C. 새 결정 (구현 전 잠금 필요)

### 결정 #16 — MatCap 마스크: 별도 샘플러 vs ILM.R (결정 #11 재검토) — **확정 ✅**
- 결정 #11은 MatCap 강도를 ILM.R 패킹으로 정했으나, 모델러가 텍스처/마스크 분리를 요구(#13).
- **확정: 별도 `_MatCapMask` 추가** + `_USE_MATCAP_MASK` 키워드. 마스크 미할당 시 **ILM.R 폴백**(키워드 off). ILM 규약 자체 불변(R = spec/matcap 강도 의미 유지).

### 결정 #17 — 음영 모델: 파라메트릭 1·2차 밴드 vs RampMap LUT — **확정 ✅**
- **확정: 파라메트릭 1·2차 밴드를 1차 제어로.** 경계 위치/블러/색을 1·2차 각각(`_ShadowBorder/_ShadowBlur/_ShadowColor`, `_Shadow2ndBorder/_Shadow2ndColor` 등) + 적용 범위. RampMap은 **옵션 경로**(`_USE_RAMP`)로 유지. SDF 컬러(#8)는 밴드 색 체계에 통합. ILM.G(경계 바이어스)·ILM.B(2차 선택)와 정합.
- 영향: ForwardToon 음영부 재작성, CBUFFER에 밴드 파라미터 추가(전 패스 레이아웃 동일 유지·SRP Batcher). 모든 Range 프로퍼티는 슬라이더+수치 입력 자동 제공(#1 충족).

### 결정 #18 — 테셀레이션 범위 (게임 최적화 상충)
- 실루엣 부드럽게(#14)지만 다수 캐릭터/모바일 비용 큼. 단일 티어라 무조건 켜면 인게임 부담.
- **권장**: **거리/품질 게이트(NiloToon S2)와 묶어 LOD형으로**, 또는 쇼케이스 한정. 우선순위 P2(후순위). 즉시 미구현, A~E 완료 후 별도 판단.

## D. #3 에디터 검증 체크리스트 (사용자 — Unity에서, 헤드리스 불가)
> 현재까지 변경분(Emission·LOBBY_HQ 제거·ShaderGUI·임포트 정책)이 실제로 컴파일/동작하는지 열린 에디터에서 확인.

1. **Console 컴파일 에러 0** (셰이더 + Editor 스크립트 4종).
2. URA 머티리얼 선택 → 인스펙터가 **CharacterToon GUI**(상단 Simple/Advanced, 폴드아웃, **lilToon 영문 라벨**, **마우스 롤오버 시 한글 툴팁**)로 표시.
3. `CharacterToon/Character` 셰이더 에셋 인스펙터에 **"SRP Batcher: compatible"**.
4. 메뉴 **Window/CharacterToon → Reimport URA Textures** 실행(마스크/SDF sRGB→linear 교정).
5. 머티리얼 핑크 없음, 씬 정상 렌더. (선택) Face SDF Validator로 음영 경계 이동 확인.
6. 결과(에러/이상) 회신 → 있으면 우선 수정 후 #2 착수.
