# 07. CharacterToon 셰이더 — 오픈소스 대조 분석 (최적화 & 퀄리티)

> 작성 2026-06-20. 우리 `CharacterToon`을 `OpenToonShaders/`의 lilToon / NiloCat URP ToonLit / UTS3 / Poiyomi와 **실제 코드 대조**해 게임 최적화 정도와 퀄리티 수준을 평가.
> 근거: 우리 셰이더 직접 측정 + NiloCat 라이팅/패스 구조 직접 확인. (lilToon/Poiyomi는 규모만 측정)

## 종합 평가

| 항목 | 평가 | 요지 |
|---|---|---|
| 게임 최적화 | **B+** | 구조·변형 규율 우수. 즉시 고칠 낭비 1건(미사용 부가광 키워드). |
| 퀄리티(기능 커버리지) | **A−** | UTS3급 캐릭터 표현. lilToon만큼 광범위하진 않으나 게임엔 충분. |
| 퀄리티(동적 라이팅 정합) | **C+** | 단일 키라이트 전제. 부가광/SSAO/리플렉션프로브 통합 약함. |

## 1. 규모/구조 대조

| 셰이더 | 성격 | 패스 | 변형 전략 |
|---|---|---|---|
| **우리** | 게임 캐릭터 단일 | 5 | `shader_feature_local` + 비구조 토글은 float 분기 |
| NiloCat | URP 예제(골격 레퍼런스) | ~5 | 풀 URP `multi_compile` |
| UTS3 | 프로덕션 애니메 | 多 | 무거움 |
| lilToon | VRChat 만능 | 多 | **변형 폭발**(65 .shader/144 inc) |
| Poiyomi | VRChat 만능 | 多 | 최대 무게(42 .shader/61 inc) |

lilToon·Poiyomi는 아바타용 만능 → 게임 런타임 부적합. 우리는 NiloCat 골격 + UTS3급 기능의 **게임 지향 단일 셰이더**. 방향성 정확.

## 2. 최적화 — 강점
1. **SRP Batcher 규율**: 단일 `UnityPerMaterial`, 매프레임 글로벌은 CBUFFER 밖, 전 패스 동일 레이아웃.
2. **변형 경제**: `shader_feature_local`로 변형이 실제 머티리얼 조합에 한정. 비구조 옵션(MatCap 마스크/블렌드/마스크타입)은 키워드 대신 float 분기 → 폭발 회피.
3. **단일 키라이트(캐릭터 라이트)**: 부가광 루프 대신 1광원 → 다수 캐릭터 ALU 절약.
4. **린한 보조 패스**: Outline이 라이팅 multi_compile 없이 가벼움(NiloCat 아웃라인은 풀 라이팅 키워드).
5. **half 우세**(half 83/float 65); 미분 TBN만 float.

## 3. 최적화 — 고칠 점
1. **🔴 미사용 부가광 키워드 = 낭비**: `_ADDITIONAL_LIGHTS_VERTEX/_ADDITIONAL_LIGHTS`, `_CLUSTER_LIGHT_LOOP`를 `multi_compile`로 선언만 하고 frag 미사용 → 변형 ×3 낭비. **루프 구현(퀄리티) 또는 키워드 제거(변형↓)**. → 2026-06-20 **키워드 제거로 정리**(부가광 셀셰이딩은 L3 별도 퀄리티 과제로 분리). [완료]
2. **🟡 S2 거리 품질 페이드 미구현**: 단일 티어 always-on의 원거리 비용 방어(P0 리스크, Docs/04). 다수 캐릭터 프로파일링 핵심.
3. ~~🟡 fog 미적용~~ → **해결(2026-06-20)**: ForwardToon·Outline에 `multi_compile_fog` + `ComputeFogFactor`(vert) + `MixFog`(frag). 안개 씬 분리감 해소. 키워드 off=no-op.
4. **🟢 미세**: MatCap 1·2가 `normalVS` 각각 재계산(공유 가능), Cutout 깊이/그림자 패스 미적용(2-3 후속).

### ⚠️ 인스턴싱에 대한 정정 (중요)
초기 권장에서 "GPU 인스턴싱 추가"를 들었으나, **재검토 결과 우리 케이스엔 효과 없음**:
- **SRP Batcher 우선순위**: SRP Batcher 호환 셰이더(=우리)는 GPU 인스턴싱보다 SRP Batcher가 우선 적용된다(둘 동시 불가). 즉 인스턴싱 pragma를 넣어도 **engage되지 않음**.
- **스키닝 메시 비호환**: 캐릭터는 SkinnedMeshRenderer → GPU 인스턴싱 자체가 적용 불가. SRP Batcher만 동작.
- **결론**: "다수 캐릭터 쿼터뷰" 비용은 **이미 SRP Batcher가 처리**(보유). 인스턴싱은 inert. → 권장 철회. 실제 perf 레버는 **SRP Batcher(보유) + S2 거리 페이드(미구현) + 부가광 정리(완료)**.

## 4. 퀄리티 — 강점
- 기능 커버리지가 UTS3에 근접: Face SDF, MatCap×2(Add/Multiply), Angel Ring, Skin SSS, Rim+Depth Rim, PBR 툰 스페큘러+환경반사, 노말맵, 아웃라인(인버티드헐+스크린스페이스), Emission, 눈 시차, Transparent, 스텐실, **퍼-캐릭터 라이트(NiloToon 차별)**, 파라메트릭 1·2차 그림자 밴드.
- NiloCat 예제보다 표현 풍부(예제는 셀+occlusion+아웃라인 수준).
- ILM 규약·프리셋·lilToon식 한글 UX로 작업자 친화.

## 5. 퀄리티 — 갭 (NiloToon/UTS3 대비)
1. ~~**부가광 셀셰이딩 없음**(L3)~~ → **해결(2026-06-20)**: `ShadeAdditionalToon` 루프 구현(Forward+클러스터 포함), `_USE_ADD_LIGHTS` 토글·`_AdditionalLightStrength`. point/spot이 캐릭터에 셀 음영+그림자. 부가광 키워드 재추가(이번엔 실사용).
2. ~~SSAO 미적용~~ → **해결(2026-06-20)**: `multi_compile_fragment _SCREEN_SPACE_OCCLUSION` + `GetScreenSpaceAmbientOcclusion`. indirect AO→앰비언트, direct AO→직접광/부가광. `_AOStrength` 다이얼(툰 과AO 방지). 키워드 off=no-op.
3. ~~리플렉션 프로브 블렌딩/박스투영 없음~~ → **해결(2026-06-20)**: `GlossyEnvironmentReflection` 5-arg 오버로드(positionWS+screenUV) + `_REFLECTION_PROBE_BLENDING`/`_REFLECTION_PROBE_BOX_PROJECTION` 키워드. 실내 이동 시 반사 튐 완화 + Forward+ 프로브 아틀라스. 키워드 off=단일프로브(무회귀).
4. **미분 TBN 한계**: 메시 탄젠트 미사용(결정 #3 양립). 극단 UV 품질 편차 가능.
5. **눈 시차 = 경량 오버레이만**(전체 홍채 깊이는 별도 쉐이더 — 3-1 결정).

## 6. 결론
- **최적화**: lilToon/Poiyomi류보다 훨씬 가벼운, 상업 게임 기준 적합 구조. **부가광 정리(완료) + S2 거리 페이드**로 다수 캐릭터에 안전. 인스턴싱은 불필요(SRP Batcher가 대체).
- **퀄리티**: 단일 키라이트 전제 스타일라이즈드 캐릭터로는 UTS3/NiloToon급. 동적 부가광·SSAO·리플렉션 통합이 약해 복잡 라이팅 씬에선 하락. 쿼터뷰/로비(라이팅 통제)엔 충분히 높음.

## 7. 우선 권장 (효과순)
1. ✅ 부가광 키워드 제거(완료). 
2. ✅ **S2 거리 품질 페이드(완료, 2026-06-20)** — `_USE_QUALITY_FADE` 키워드 + `_QualityFadeStart/End`. 원거리에서 비싼 가산 HQ항(MatCap×2/PBR 스펙·반사/AngelRing/SSS/DepthRim)을 `if(hqAmount>0)`로 **연산·샘플 스킵 + 페이드**(팝 방지). hqAmount는 **오브젝트 원점 거리**로 계산해 드로우 전체 uniform(쿼드 발산·파생연산 문제 없음, 캐릭터 단위 페이드). 키워드 off=완전 no-op. Codex 검증(초안 (b) FAIL→오브젝트거리로 수정→전항목 PASS).
3. 부가광 셀셰이딩 L3 — 동적 라이팅 퀄리티(원하면 별도 구현).
4. ✅ fog 적용, SSAO 적용, 리플렉션 프로브 블렌딩 — URP 통합 정합 **(2026-06-20 전부 완료)**.
5. MatCap normalVS 공유 등 미세 정리.
- ❌ GPU 인스턴싱 — SRP Batcher 우선 + 스키닝 비호환으로 inert. 미진행.
