## 0. 요약 — 캐릭터 셰이더와 무엇이 다른가

배경은 캐릭터 셰이더의 일부 기능을 **빼고**, 배경 고유 요구를 **더한다**. 핵심 차이만 추리면:

| 구분 | 캐릭터(`CharacterToon`) | 배경(`SceneToon`) |
|---|---|---|
| 라이팅 소스 | 캐릭터 전용 라이트 리그 글로벌 주입 | **씬 메인 라이트 + 베이크 GI/라이트맵**(정적이 대부분) |
| 음영 | NdotL Ramp + Face SDF | Ramp/밴드 음영 **공유** + **라이트맵 포스터라이즈**(SDF·SSS·Angel Ring 없음) |
| 그림자 | 주로 그림자 캐스트 | **캐릭터·환경 그림자 수신이 1순위** + 자기 그림자 |
| 아웃라인 | 인버티드 헐(스무스 노멀 베이크) | **스크린스페이스(풀스크린) 재사용** — 하드서피스 열린 엣지의 헐 틈 회피 |
| 변형 | 부위 키워드(Face/Hair/Skin/Cloth) | **타입 키워드(Static/Foliage)** + 지형/스카이는 별도 셰이더 |
| 고유 기능 | MatCap·Angel Ring·SSS·Eye | **바람 애니메이션·잎 투과(translucency)·트라이플래너·정점컬러 블렌딩** |
| 비용 성향 | 소수 고비용 | **다수·대면적 저비용**(드로 수·오버드로 민감) |

**전략적 결론**: 캐릭터 라이팅 코어(half-Lambert→Ramp, 파라메트릭 밴드, 림)를 공유 인클루드로 끌어올려 룩을 일치시키고, 배경은 그 위에 라이트맵 툰화 + 타입별 기능만 얹는다. 아웃라인은 이미 풀스크린이므로 배경은 **DepthNormals만 제대로 출력하면 거의 공짜로** 동일 외곽선에 참여한다.

## 1. 구현 목표 / 사용 시나리오

- **쿼터뷰 인게임**: 넓은 맵·다수 배경 오브젝트. 드로콜·오버드로·라이트맵 메모리가 곧 비용 → SRP Batcher 친화 + GPU instancing 친화가 최우선.
- **1인칭 로비 쇼케이스**: 근접 배경(소품·벽·바닥)도 캐릭터와 같은 셀 톤으로 보여야 위화감이 없다. 근접에서 라이트맵 밴딩/계단이 튀지 않을 것.
- **공통**: 캐릭터와 배경의 **그림자·외곽선·셀 톤이 한 화면에서 동일 언어로** 읽혀야 한다(시각 통일이 본 셰이더의 존재 이유).

## 2. 셰이더 구조

캐릭터의 "단일 셰이더 + 로컬 키워드" 철학을 따르되, **타입별 정점 파이프라인 차이가 큰 것은 분리**한다. (정적/식생은 프래그먼트가 거의 같고 정점 단계만 다르므로 1개로 통합, 지형/스카이는 시스템 통합 요건이 달라 분리.)

```text
Assets/SceneToon/
├─ Shaders/
│  ├─ SceneToon.shader          # 정적 소품/건축물 + 식생 (타입 키워드로 분기)
│  ├─ SceneToon_Terrain.shader  # Unity Terrain 통합(스플랫/트라이플래너)  — D-4
│  └─ SceneToon_Sky.shader      # 절차적 그라데이션/큐브맵 unlit          — D-7
└─ ShaderLibrary/
   └─ (공유 코어는 ToonShared로, 3장 참조)
```

`SceneToon.shader` 패스 구조 (캐릭터와 동형으로 맞춰 렌더러 피처 호환):

```text
SceneToon.shader
├─ SceneForward   LightMode = UniversalForward (베이크 GI/추가광 수신, 캐릭터 #1과 정합 — D-1)
├─ ShadowCaster   URP 표준 (배경끼리/캐릭터에 그림자 투사)
├─ DepthOnly      URP 표준
├─ DepthNormals   필수 — 스크린스페이스 아웃라인 엣지 검출에 배경이 참여하는 핵심
└─ (Meta)         라이트맵 베이크용 Meta 패스
```

타입/기능 로컬 키워드:

```hlsl
#pragma shader_feature_local _ _SCENE_TYPE_FOLIAGE          // 기본(off)=Static
#pragma shader_feature_local _ _USE_WIND                    // 식생 바람 (정점)
#pragma shader_feature_local _ _USE_TRANSLUCENCY            // 잎 투과(빛 통과)
#pragma shader_feature_local _ _ALPHATEST_ON                // 알파 클립(식생 컷아웃)
#pragma shader_feature_local _ _USE_VERTEXCOLOR_BLEND       // 정점컬러 2-알베도 블렌딩
#pragma shader_feature_local _ _USE_TRIPLANAR               // 트라이플래너(바위/절벽 UV 스트레치 제거)
#pragma shader_feature_local _ _USE_NORMALMAP
#pragma shader_feature_local _ _USE_OUTLINE_SUPPRESS        // per-object 외곽선 억제 마스크
// URP 멀티컴파일: 라이트맵/그림자/추가광은 패키지 표준 multi_compile 사용
#pragma multi_compile _ LIGHTMAP_ON
#pragma multi_compile _ DIRLIGHTMAP_COMBINED
#pragma multi_compile _ _MAIN_LIGHT_SHADOWS _MAIN_LIGHT_SHADOWS_CASCADE _MAIN_LIGHT_SHADOWS_SCREEN
#pragma multi_compile _ _ADDITIONAL_LIGHTS_VERTEX _ADDITIONAL_LIGHTS
#pragma multi_compile_fragment _ _SHADOWS_SOFT
#pragma multi_compile _ _FORWARD_PLUS
```

> 실제 사용 조합은 **머티리얼 프리셋**으로 고정(StaticOpaque / FoliageCutout / Rock_Triplanar / Terrain …)해 변형 수를 억제한다(캐릭터 본편 8장 원칙 계승).

## 3. 배경 셀 코어 라이브러리 (ToonShared — 배경 전용, D-3 재검토)

배경이 캐릭터와 **같은 셀 톤**을 내려면 음영 함수가 같아야 한다. 단 **캐릭터 셰이더를 여기에 의존시키지는 않는다**(아래 ⚠ 참조) — ToonShared 는 배경 셰이더(SceneToon + Terrain)가 공유하는 셀 코어이며, 그 수식은 CharacterToon 의 인라인 셀 수식을 복제해 톤을 맞춘다.

제안 구조:

```text
Assets/ToonShared/ShaderLibrary/
├─ ToonRamp.hlsl        # SampleILM, half-Lambert→Ramp U, _RampRow 샘플
├─ ToonBandShadow.hlsl  # 파라메트릭 1·2차 밴드(_ShadowBorder/_ShadowBlur/_Shadow*Color)
└─ ToonRim.hlsl         # 프레넬 림(step/smoothstep 선택)
```

- **⚠ D-3 재검토(2026-06-23, 결정 변경): 캐릭터는 ToonShared 에 의존하지 않는다(안정성).** 성숙·시각검증 대기 중인 캐릭터 셰이더에 신생 배경 셰이더의 변경이 흘러들 통로를 만들지 않기로 함. ToonShared 는 **배경 전용 셀 코어**(SceneToon + SceneToon_Terrain 가 공유)이며, 그 수식은 **CharacterToon 인라인 셀 수식을 복제**한 것이라 한 화면 톤이 일치한다.
- **톤 일치 유지 방식**: 컴파일 타임 단일 출처(공유 #include)가 아니라 **복제 + 수동 동기화**. 캐릭터의 셀 밴드/half-Lambert 수식이 바뀌면 ToonShared 를 맞춰 동기화(각 ToonShared 파일 헤더에 동기화 주석 명시). 캐릭터 수식 자체가 안정적이라 동기화 빈도는 극히 낮음.
- ILM 채널 규약(R=스펙/MatCap, G=그림자 편향, B=폭/보조램프, A=내부선/외곽억제)은 **프로젝트 전역 불변**. 배경에서 ILM을 쓰면 같은 의미를 따른다(주로 G=그림자 편향, A=외곽선 억제만 사용). ILM 누락 시 중립값 `(0, 0.5, 0.5, 0)`.
- ILM 채널 규약(R=스펙/MatCap, G=그림자 편향, B=폭/보조램프, A=내부선/외곽억제)은 **프로젝트 전역 불변**. 배경에서 ILM을 쓰면 같은 의미를 따른다(주로 G=그림자 편향, A=외곽선 억제만 사용). ILM 누락 시 중립값 `(0, 0.5, 0.5, 0)`.

## 4. 데이터 규약 (배경)

### 4.1 텍스처

| 텍스처 | 용도 | 정적 | 식생 | 지형 |
|---|---|---:|---:|---:|
| BaseMap | 기본 색(식생은 알파=컷아웃) | 필수 | 필수 | 레이어별 |
| BumpMap | 노멀 | 선택 | 선택 | 레이어별 |
| ILMMap | 그림자 편향(G)/외곽 억제(A) — 선택 | 선택 | 선택 | OFF |
| MaskMap | (정적) AO/Smoothness/Detail 패킹 | 선택 | OFF | OFF |
| WindNoise | (식생) 바람 위상 노이즈 | OFF | 선택 | OFF |
| SplatMap | (지형) 레이어 가중 | OFF | OFF | 필수 |

- **라이트맵 UV(UV1)** 와 정적 배칭/GI 메타가 정상 동작하도록 Meta 패스 유지. **정점컬러**는 식생 바람 가중·2-알베도 블렌딩에 사용(D-5).
- RampMap은 배경도 캐릭터와 동일 슬롯/규약. 부위 행 대신 "재질군 행"(돌/나무/천/금속)을 둬도 됨.

### 4.2 SRP Batcher / 인스턴싱

- 모든 머티리얼 프로퍼티는 `CBUFFER_START(UnityPerMaterial)`에, **전 패스 동일 레이아웃**. 캐릭터와 동일 규율.
- 매 프레임 주입 글로벌(씬 라이트는 URP가 제공하므로 추가 주입 거의 없음)은 CBUFFER 밖.
- 다수 동일 메시(나무·돌)는 **GPU Instancing/`DOTS_INSTANCING` 호환** 유지(식생 대량 배치 대비). `MaterialPropertyBlock` 남용 금지.

## 5. 라이팅 — 배경 핵심

### 5.1 베이크 GI 툰화(라이트맵 포스터라이즈)

배경은 대부분 라이트맵 베이크다. 라이트맵을 그대로 쓰면 PBR스럽게 부드러워 셀 톤이 깨진다 → **라이트맵 결과를 Ramp/밴드로 양자화**한다.

```hlsl
// 1) 라이트맵(또는 SH) 디퓨즈 GI 취득
half3 bakedGI = SAMPLE_GI(input.lightmapUV, input.vertexSH, N);
// 2) 밝기를 밴드로 양자화해 셀 톤화 (직접광 밴드 공식과 동일 라이브러리 사용)
half giLum = Luminance(bakedGI);
half giBand = ToonBand(giLum, _GIBandCount, _GIBandSoftness); // ToonShared
half3 toonGI = bakedGI * lerp(_GIShadeColor.rgb, 1.0h, giBand);
```

- `_GIBandCount`/`_GIBandSoftness`로 근접(로비)에서 밴딩 강도 조절 → 근접 계단 방지(D-2).
- 직접광은 캐릭터와 **동일** 밴드 음영(`ToonBandShadow.hlsl`)을 태워 한 화면 톤 일치.

### 5.2 그림자 수신(1순위)

- 메인 라이트 그림자 + 추가광 그림자 수신. 배경이 캐릭터 그림자를 받는 게 셀 룩 합성의 핵심.
- 캐릭터가 배경에 드리우는 그림자가 **밴드 경계와 충돌해 이중선**으로 보이지 않게, 그림자 항을 밴드 *입력*에 합산(곱이 아니라 ndotl 편향)하는 방식 검토(D-2와 함께).

### 5.3 추가광 / Forward+

- 횃불·네온 등 점/스폿 광원을 배경이 셀 음영으로 받게 `_ADDITIONAL_LIGHTS` 경로 지원. Forward+ 클러스터 경로 동작 확인(캐릭터 본편 결정 #8과 동일 환경이므로 그 결론 재사용).

## 6. 아웃라인 — 스크린스페이스 재사용 (기준)

핵심: **기존 `Hidden/CharacterToon/ScreenSpaceOutline` + `CharacterToonRendererFeature`는 이미 풀스크린**(`AfterRenderingOpaques`, depth/normal 엣지). 배경은 다음만 충족하면 별도 아웃라인 패스 없이 동일 외곽선에 참여한다.

1. **DepthNormals 패스 출력**(2장) — 노멀 불연속 엣지 검출 대상이 됨.
2. depth/normal 임계 튜닝 — 배경은 면적·깊이 폭이 커서 캐릭터용 임계와 다를 수 있음 → 렌더러 피처 세팅을 씬별로 조정(또는 캐릭터/배경 레이어 분리 검토).

추가 옵션(필요 시):
- **per-object 외곽선 억제**: ILM.A 또는 `_USE_OUTLINE_SUPPRESS` 마스크를 스텐실에 기록해 SS 아웃라인이 특정 배경물(예: 먼 산, 스카이접합)에서 선을 빼게 함.
- **영웅 소품 한정 인버티드 헐**: 클로즈업 핵심 소품만 캐릭터식 정점 외곽선 옵션 패스 제공(하드서피스 열린 엣지 틈은 스무스 노멀 베이크 필요 → 비용 큼, 기본 비활성). 결정 D-6.
- 스카이/배경막은 아웃라인 비대상(깊이 최후단 또는 stencil 제외).

## 7. 타입별 기능

### 7.1 정적 소품/건축물 (기본)
- Ramp/밴드 음영 + 라이트맵 툰화 + 노멀 + (선택)Detail/AO. 트라이플래너 옵션(바위·절벽 UV 스트레치 제거, D-4와 공유).
- 정점컬러 2-알베도 블렌딩(이끼·오염·눈 쌓임 표현).

### 7.2 식생 (나무·풀·잎)
- **알파 컷아웃**(`_ALPHATEST_ON`) + 양면(`Cull Off`) 렌더. 컷아웃 가장자리 디더/알파-투-커버리지 옵션.
- **바람 애니메이션**(`_USE_WIND`): 정점에서 `WindNoise`+정점컬러 가중(R=주간선 강성, G=잎 펄럭) 사인 합성. Unity `WindZone` 연동 vs 셰이더 단독은 D-5.
- **잎 투과**(`_USE_TRANSLUCENCY`): 뒤에서 빛 받을 때 `VdotL` 기반 밝힘(반밀도 backlight). 셀 톤이라 부드러운 SSS 대신 1~2밴드 단순화.

### 7.3 지형 — Unity Terrain + 메시 지형 동시 지원 (D-4 확정)
**두 입력 경로를 같은 음영 코어에 연결**한다(아트팀이 Unity Terrain 툴과 메시 지형을 혼용하므로):
- **Unity Terrain 경로** (`SceneToon_Terrain.shader`): `TerrainLit` 구조 규약(레이어/스플랫맵/베이스맵 GI) 준수 + 스플랫 레이어별 Ramp 음영.
- **메시 지형 경로** (`SceneToon.shader`의 `_USE_TRIPLANAR` + 정점컬러 블렌딩): 메시로 만든 지형/절벽에 트라이플래너로 경사 UV 스트레치 제거, 정점컬러/높이 기반 레이어 블렌딩.
- 두 경로 모두 **3장 공유 라이브러리(밴드/Ramp/라이트맵 툰화)를 그대로 호출** → 지형 제작 방식과 무관하게 톤 일치.

### 7.4 스카이/배경막 (`SceneToon_Sky.shader`) — D-7
- 라이팅 무관 unlit. 절차적 2~3색 그라데이션(천정/지평/지면) 또는 큐브맵. 깊이 최후단, 아웃라인 비대상.
- 기존 URP 스카이박스를 유지하고 본 셰이더는 "배경막 평면/돔"만 담당하는 안도 검토.

## 8. SRP Batcher와 변형 관리

- 캐릭터 본편 8장 규율 그대로: 단일 CBUFFER 레이아웃, 전 패스 동일.
- 타입 기능은 `shader_feature_local`, 실사용 조합은 **머티리얼 프리셋**으로 고정.
- 빌드 전 `ShaderVariantCollection` + URP variant stripping 확인. 배경은 라이트맵×그림자×추가광 멀티컴파일이 곱해져 변형이 커지기 쉬우므로 **스트리퍼(캐릭터 `CharacterToonShaderStripper` 패턴 재사용)로 미사용 조합 제거**.

## 9. 검증 툴 (최소)

배경 셰이더 정확성 게이트:
1. **라이트맵 툰화 프리뷰**: 베이크 전/후, 밴드 수 변화에서 근접 계단·톤 일치 점검(B1 게이트).
2. **아웃라인 참여 검증**: 배경 DepthNormals 출력 → SS 아웃라인 엣지가 배경/캐릭터에 일관히 그려지는지(B3 게이트).
3. **식생 바람 프리뷰**: 무풍/강풍에서 정점 폭주·정규화 깨짐 없는지(B2).
4. **합동 시각검증**: `09_합동_시각검증_체크리스트.md`에 배경 항목 추가(캐릭터+배경 한 화면 톤 일치).

## 12. 완료 기준

- 대표 배경 씬에서 정적물·식생·지형·스카이가 **캐릭터와 동일한 셀 톤·그림자·외곽선 언어**로 한 화면에 어우러진다.
- 배경이 캐릭터 그림자를 받고, 캐릭터/배경이 **같은 스크린스페이스 외곽선**에 일관 참여한다.
- 라이트맵 기반 정적 배경이 SRP Batcher/Instancing 친화 변형 수로, 인게임 대면적·로비 근접 목표 프레임을 만족한다.
- 미해결 결정 레지스터(D-1~D-9) 전 항목이 해당 마일스톤에서 닫힌다.
- 공유 라이브러리 추출 후 **캐릭터 셰이더 회귀가 없음**을 B5에서 확인한다.
